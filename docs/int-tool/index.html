    input

    • 123 — decimal integer (parsed as an int).
    • 0xFF — hexadecimal integer (prefix 0x).
    • 0o777 — octal integer (prefix 0o).
    • 1.5 .5 1. — decimal float (parsed as an f64).
    • 1e10 1.5e-3 — scientific notation float (parsed as an f64).
    • 1.5f 3.14F — float with f/F suffix is parsed as an f32.
    • 1.0l 1.0L — float with l/L suffix is parsed as an f64.
    • 42f — a bare integer with f is treated as an f32.
    • An optional + / - sign may precede any number.

    If the input can't be parsed, the box turns red and the last result stays.

    A visualizer for the bit-level representation of integers and floats. Type a number, pick a data structure, and inspect how its bits are laid out and reinterpreted.

    binary visualization

    • Each column is one bit; the index is shown underneath (bit 0 at the right).
    • Bits are grouped into bytes (8 bits) with spacing between groups.
    • The bit width is chosen by the data structure (or the number's highest set bit).
    • Click any bit to toggle it — the value updates and the input box follows.

    data structure

    • Choose a structure (u/i 8–64, f32, f64) to reinterpret the bits.
    • Integer: the sign bit is highlighted.
    • Float: sign (red), exponent (blue), and fraction (green) bits are highlighted.
    • auto (checked by default) picks a fitting structure as you type. Uncheck to lock it.

    representation

    • Shows decimal, hex, octal (integers), exponent/fraction fields (floats), and little/big endian byte dumps.
